Groff

Why do I use groff?

To be honest I have been using groff for a couple of days. However, there are two benefits I have noticed:

  1. Compilation speed
  2. Clean “code”

Compilation speed

The benefit of compilation speed should be clear to anyone that has dealt with a medium sized Latex document. The compilation times are horrendous. Because groff is more minimal by trait, compilation times are faster by a large margin. However, it is to be noted that I haven’t created any large documents with groff yet.

Clean “code”

Although, not comparable to Markdown, groff syntax is pretty clean. Most macros have to start at the line beginning, which makes the code much cleaner. “Bolding” and “Italicing” can be done inline with "\fB" and "\fI". Depending on the macro set being used section heading are very comparable to Markdown headings. For example: “.sh [depth] [title]” for the me-macros.

Macros

There are a number of macro-sets that are installed by default: me, mm, mom, ms.

In the beginning it might be very difficult to choose the right macro-set for you. If you are first getting started to use groff, the ms macros are probably the easiest to get started with. The reason for that is, that there are a bunch of resources about it online.

However, I found that the me macros are way cleaner and almost have the same features. I will probably follow up this post with an me tutorial.

Preprocessors

The power of groff comes from the multitude of preprocessors available. The most important pre-installed ones are: eqn, refer, tbl and pic.

  1. eqn: equations support
  2. refer: bibliography support (similar to bibtex, etc…)
  3. tbl: table support
  4. pic: picture compiler (similar to tikz)

Without preprocessors groff would probably be useless. However, with them it is almost on par with Latex.

Problems

Using custom fonts is kind of difficult.

There is no “real” utf-8 support. The only way utf-8 is supported is via a pre-processor (preconv) that converts utf-8 to characters available in groff.

The last 2 problems arise from groff being single-pass:

  1. Forward references aren’t available by default (in most macro-sets).
  2. A Table of contents can only be added to the end of the file
